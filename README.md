# Capstone Project

A software engineering capstone repository built with modern web technologies, version control best practices, and AI-assisted development workflows.

---

## Project Overview

This project serves as the foundation for a capstone application developed as part of software engineering coursework. The repository is structured to support a full-stack web application using HTML5, CSS3, JavaScript (ES6+), and Node.js.

Currently in its initial setup phase, the repository establishes professional development standards—including documentation, licensing, and Git workflow conventions—before application features are implemented. The goal is to deliver a maintainable, well-documented project that demonstrates both technical implementation and software engineering discipline.

---

## Features

### Repository & Workflow

- **Professional project structure** — Organized layout with documentation, licensing, and environment configuration
- **Conventional Commits** — Standardized commit messages for a clear and searchable Git history
- **AI-assisted development** — Cursor AI integration via `CLAUDE.md` for consistent coding guidance
- **MIT License** — Open-source licensing for transparency and reuse

### Planned Application Capabilities

- Responsive front-end built with HTML5, CSS3, and modern JavaScript
- Node.js runtime for server-side logic and tooling
- Modular, reusable code following ES6+ standards
- Environment-aware configuration with `.gitignore` for secrets and build artifacts

## Getting Started

### Prerequisites

- [Git](https://git-scm.com/downloads)
- [Node.js](https://nodejs.org/) (LTS recommended)

### Installation

1. Clone the repository:

   ```bash
   git clone https://github.com/<your-username>/capstone-project.git
   ```

2. Navigate into the project directory:

   ```bash
   cd capstone-project
   ```

3. Install dependencies (when a `package.json` is added):

   ```bash
   npm install
   ```

## Git Workflow

This project follows the [Conventional Commits](https://www.conventionalcommits.org/) specification.

| Prefix | Purpose |
|--------|---------|
| `feat:` | Add a new feature |
| `fix:` | Resolve a bug |
| `docs:` | Update documentation |
| `style:` | Formatting or styling changes |
| `refactor:` | Improve code structure without changing behavior |
| `test:` | Add or update tests |
| `chore:` | Maintenance tasks |

**Example:**

```bash
git commit -m "feat: add user authentication module"
```
